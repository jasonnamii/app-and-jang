# α7. 각주 자동화 엔진

## 핵심
**처분례·판례·벤치마크·시나리오 4개 코퍼스 통합 인덱싱** → 약관 문구에 각주·리스크스코어·과징금 기대값 자동 삽입. 김앤장 의견서를 약관에 인라인 직조.

## 아키텍처

```
[입력] 약관 문구 (1조항·1문장 단위)
   ↓
[1단계] 키워드·의미 매칭 (4 코퍼스)
   ├─ α1 처분례 → 처분 빈도·평균 과징금
   ├─ α2 판례 룰엔진 → 무효확률
   ├─ α3 벤치마크 DB → 유사 서비스
   └─ α6 분쟁 시나리오 → 패배 시나리오·가드레일
   ↓
[2단계] 점수 합산
   ├─ 리스크스코어 (低/中/高/극高)
   ├─ 과징금 기대값 E[X] = Σ Pi × Ki
   └─ 무효확률 (룰엔진 출력)
   ↓
[3단계] 각주 합성
   ├─ [근거] 법조문
   ├─ [리스크] 점수 + 과징금 기대값
   ├─ [처분례] Top 1~3건
   ├─ [판례] Top 1~3건
   ├─ [벤치마크] 유사 사례
   └─ [가드레일] 권장 수정 문구
   ↓
[출력] 문구 + 인라인 각주 (마크다운)
```

## 출력 포맷 표준

```markdown
제3조(개인정보 보유기간) 회원 탈퇴 시 즉시 파기하며,
관계 법령에 따라 보관이 필요한 경우 해당 기간 동안 보관합니다.
> [근거] 개인정보보호법 §21, §39의6
> [리스크] 低 · 과징금 기대값 ~300만원 [신뢰도: 상]
> [처분례] PIPC_2024_0317 (corpus_id 검증가능)
> [판례] CASE_003 서울중앙지법 2023가합XXXXX
> [벤치마크] 토스 PP §4 (2026-02 개정) — 동일 표현
> [무효확률] 35%
> [가드레일] "탈퇴 후 30일 이내 파기" 명시 권장
```

## Hallucination 방어 (5축)

| 위험 | 방어 |
|------|------|
| 가짜 사건번호 생성 | corpus 미등록 번호 출력 ✗ (whitelist 강제) |
| 가짜 판례 인용 | corpus_id 필수 (footnote_validator.py 검증) |
| 잘못된 과징금액 | 코퍼스 메타데이터에서만 직접 읽기, LLM 생성 ✗ |
| 비공식 출처 | 신뢰도 등급(상/중/하) 표기 |
| 옛 법령 인용 | 코퍼스 갱신일 + 법령 시행일 매칭 |

**최종 방어선:** 모든 출력에 `(corpus_id: {ID})` 포함 → 사용자 즉시 원문 검증.
